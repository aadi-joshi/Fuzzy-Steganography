# Data Directory

This folder holds experiment inputs and outputs for the adaptive fuzzy LSB steganography framework.

## Layout

```
data/
├── BOSSbase_1.01/          # Real benchmark (download required)
├── BOWS2/cover/            # Real benchmark (download required)
├── mirflickr/              # Real benchmark (download required)
├── covers_v2/              # Synthetic covers (generated by pipeline)
├── outputs/                # V1 legacy experiment CSVs
├── outputs_v2/             # Primary synthetic experiment results (committed)
├── outputs_bossbase/       # BOSSBase validation results (committed)
├── outputs_bows2/          # BOWS2 validation results (committed)
├── outputs_mirflickr/      # MIRFLICKR validation results (committed)
└── .downloads/             # Cached archives (created by download script, gitignored)
```

## Datasets

### 1. Synthetic corpus (primary evaluation) — generated, not downloaded

| Property | Value |
|----------|-------|
| Count | 1,000 images |
| Size | 256 x 256 RGB PNG |
| Categories | smooth, noise, natural, textured, mixed (200 each) |
| Seed | 42 |
| Generated by | `experiments/generate_dataset.py` (Stage 1 of V2 pipeline) |
| Output path | `data/covers_v2/` |

This dataset is **created by the repository**, not downloaded. It is fully reproducible:

```bash
python experiments/run_v2.py --config config/config_v2.yaml --stage 1
```

No Kaggle or external hosting is needed: any reviewer with Python and the repo can regenerate the identical 1,000 images from the fixed seed and documented generator functions.

## Automated download

From the repository root:

```bash
pip install -r requirements.txt
python scripts/download_datasets.py --dataset all
```

Per dataset:

```bash
python scripts/download_datasets.py --dataset bossbase
python scripts/download_datasets.py --dataset bows2
python scripts/download_datasets.py --dataset mirflickr
```

Preview without downloading:

```bash
python scripts/download_datasets.py --dataset all --dry-run
```

---

## Manual download (if the script fails)

1. Download the archive from the verified URL above.
2. Extract to a temporary folder.
3. Copy image files into the expected path:
   - BOSSBase: `data/BOSSbase_1.01/*.pgm`
   - BOWS2: `data/BOWS2/cover/*.pgm`
   - MIRFLICKR: `data/mirflickr/*.jpg`
4. Ensure at least 200 images are present. The experiment runner uses the first 200 files in **sorted filename order**.

## Image selection protocol

The experiment code (`experiments/run_real_dataset.py`) selects the first `n_images` files in **sorted filename order** from the dataset directory (default `n_images: 200`). This is deterministic given the same extracted archive contents.

Note: some paper prose refers to a "random 200-image subset." The committed code and results use sorted filename order, not random sampling. Reproduction should follow the code behavior above.
